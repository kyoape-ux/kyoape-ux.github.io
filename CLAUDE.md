## 檔案清單與保護等級

以下檔案皆為已開發完成的正式功能，**禁止覆蓋、替換或縮減**：

| 檔案 | 用途 | 狀態 |

| `index.html` | 工具站首頁（入口） | 已上線 |

## 重要規則

1. **不可用佔位頁面覆蓋已完成的檔案** — 每個 HTML 檔案都是獨立開發的完整功能，修改前必須先讀取確認現有內容
2. **修改前先 `git diff`** — 提交前確認變更範圍合理，避免整檔替換
3. **不可在 sync / 整理 commit 中大幅刪減其他檔案** — 只改你正在開發的檔案
4. **commit + push 不需詢問** — 修改完直接推送部署

## 部署方式

- GitHub Pages（從 `main` 分支根目錄部署）
- 推送到 `main` 即自動部署
